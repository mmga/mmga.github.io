git学习笔记

发表于 2016-01-13

相当零散一笔记。

之前一直野路子，只用Android Studio和github桌面客户端里面最基础的功能，高级点的就不敢碰了，怕把存档搞坏。。今天仔细照着-廖雪峰git教程-敲吧了一遍，才发现原来根本没什么神秘的嘛~这里记录下一些基本的操作，方便查询

add, commit, status, 已敲烂。

查看：
git log 可以查看提交历史，以便确定要回到哪个版本。
git log --graph 可查看分支图。
git log --pretty=oneline 避免看花眼。
git diff 查看修改内容。
git reflog 查看命令历史，以便确定要回到未来的哪个版本。

撤销：
git checkout -- <name>把文件在工作区的修改全部撤销，就是让他回到最近一次commit或add的状态。
git reset HEAD <name>add的逆操作，把缓存区撤销。
git reset --hard HEAD^ HEAD^表示上个版本，HEAD^^大上个版本，或者用git reflog查询id 直接回到某个版本。
git revert [commit] 新建一个commit,用来撤销指定commit,后者所有的变化都将被前者抵消，并应用到当前分支。

分支相关:
git check out -b dev 创建并切换到dev分支。等于下面两条指令。
git branch dev 创建
git checkout dev 切换
git merge dev 合并dev至当前分支。FastForward模式。看不出曾经做过合并。
git merge --no-ff -m "description" 不使用ff模式合并，merge之后能看出来曾经做过合并。
git branch 列出所有本地分支
git branch -r 列出所有远程分支
git branch -d dev 删掉dev分支
git branch -D <name> 丢弃一个没被合并的分支，强行删除。
git stash 保存工作现场。修复bug时，我们会通过创建新的bug分支进行修复，然后合并，最后删除。当手头工作没完成时，先把工作现场git stash一下然后去修复bug，修复后再git stash pop回到工作现场。
git stash pop 回到工作现场。
git stash apply 恢复。
git stash drop 删掉stash内容。

远程库：
git push origin master 把本地master分支推送至github。
git remote add origin https://github.com/mmga/LearnGit.git 关联远程库。
git push -u origin master 第一次推送master分支。
git clone git@github.com:mmga/LearnGit.git 克隆到本地
git remote -v 查看远程库信息。
git pull 取回远程仓库的变化，并与本地分支合并。
git push origin branch -<name> 从本地推送分支，如果推送失败，先用git pull抓取远程的新提交。
git checkout -b <branch-name> origin/<branch-name> 创建远程origin的分支到本地.
git branch --set -upstream <branch-name> origin/<branch-name> 建立本地分支和远程分支的关联。

标签：
git tag <tagname> 新建标签，默认为HEAD，也可以在后面置顶一个commit id。
git tag -a<tagname> -m "description" 带描述。
git tag -s<tagname> -m "description" 用PGP签名标签。
git tag 查看所有标签。
git push origin <tagname> 推送一个本地标签。
git push origin --tags 推送所有未推过的本地标签。
git tag -d <tagname> 删除一个本地标签。
git push origin :refs/tags/<tagname> 删除远程标签。

配置：
git的设置文件为.gitconfig,用户主目录下是全局配置，项目目录下是项目配置
git config --list 显示当前的git配置
git config [--global] user.name 设置提交代码时的用户信息（加上global为全局，不加只设置当前项目）
git config [--global] user.email

等看了 http://git-scm.com/book/zh/v2 再补充。
